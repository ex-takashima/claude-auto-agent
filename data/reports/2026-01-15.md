## AI最新ニュース

### Anthropic公式

- **Introducing Labs**
  Anthropicは、Claude AIの最先端機能を中核とした実験的製品の開発に特化した新しいチーム「Labs」の拡大を発表しました。Instagramの共同創業者Mike Krigerが最高製品責任者からLabsに異動し、Ben Mannと共に先進的なAI機能の開発に取り組みます。同社は「速度が加速するAIの進化に対応するため、異なるアプローチが必要」とし、Claude Code、Model Context Protocol、Skills、Coworkなどの成功事例を基盤としています。Ami Voraが新たに製品組織を統括し、CTO Rahul Patilと協力して、数百万ユーザーのニーズに応える製品の拡大を目指すと述べられています。

  🔗 https://anthropic.com/news/introducing-anthropic-labs

- **Advancing Claude in healthcare and the life sciences**
  Anthropicは2026年1月、Claude for Healthcare と Claude for Life Sciences の拡張機能を発表しました。ヘルスケア部門では「CMS Coverage Database」「ICD-10」「National Provider Identifier Registry」などのコネクタを追加し、保険認可審査や請求管理を効率化します。ライフサイエンス部門では「Medidata」「ClinicalTrials.gov」「bioRxiv & medRxiv」などのプラットフォームに接続でき、臨床試験管理と規制業務を支援します。新型の「Claude Opus 4.5」は医学ベンチマークで大幅な性能向上を実現し、個人の健康データ統合機能も提供開始予定です。

  🔗 https://anthropic.com/news/healthcare-life-sciences

### GitHub Blog

- **Community-powered security with AI: an open source framework for security research**
  GitHubセキュリティラボは、AI技術とModel Context Protocol（MCP）インターフェースを活用した「GitHub Security Lab Taskflow Agent」というオープンソースフレームワークを発表しました。このツールは、セキュリティ研究者がYAML形式のタスクフローを使用して、自動化された脆弱性分析を実施できるようにします。デモでは、セキュリティ勧告から脆弱性を特定し、類似するバグを検出する方法を示しています。このフレームワークはCodeQLなどの既存セキュリティツールと統合でき、コミュニティ主導のセキュリティ向上を目指しています。

  🔗 https://github.blog/security/community-powered-security-with-ai-an-open-source-framework-for-security-research/

- **What AI is actually good for, according to developers**
  このGitHubのブログ記事は、開発者の観点からAIツールの最適な使用方法について論じています。主なポイントは以下の通りです：AIツールは開発者の「フロー状態」を保つべきで、チャット形式のインタラクションより文脈に基づいた提案が有効であること、そしてAIは「自動化のための自動化」ではなく、「開発者の判断を支援する」ものとして設計されるべきということです。記事では、開発者の生産性向上と創造性維持のバランスを強調し、「つまらない作業は自動化しながらも、アーキテクチャ判断やビジネスロジックは人間が主導権を持つべき」と述べられています。

  🔗 https://github.blog/ai-and-ml/generative-ai/what-ai-is-actually-good-for-according-to-developers/

- **Want better AI outputs? Try context engineering**
  GitHub Copilot からより良い出力を得るために、「コンテキストエンジニアリング」という新しいアプローチが注目されています。これは従来のプロンプトエンジニアリングの進化版で、AI に「正しい情報を正しい形式で提供する」ことに焦点を当てています。具体的には 3 つの実践的手法が紹介されています。第一に「カスタム指示」により、コーディング規約や命名標準などのルールを定義します。第二に「再利用可能なプロンプト」で、コードレビューやテスト生成といった頻繁なタスクを標準化します。第三に「カスタムエージェント」で、API 設計審査や セキュリティ分析など特定の役割を持つ AI アシスタントを構築します。

  🔗 https://github.blog/ai-and-ml/generative-ai/want-better-ai-outputs-try-context-engineering/

- **Light waves, rising tides, and drifting ships: Game Off 2025 winners**

### Claude Code GitHub Releases

- **v2.1.7**
  Claude Code v2.1.7では、ユーザーインターフェースと安定性の改善が複数実施されました。主な追加機能として、ターン処理時間の表示を非表示にする`showTurnDuration`設定、タスク通知にエージェントの最終応答をインライン表示する機能、パーミッションプロンプト時のフィードバック機能が含まれています。セキュリティ面では、ワイルドカードパーミッションルールの脆弱性を修正し、Windows環境でのファイル変更エラーやバッシュコマンド実行の問題を解決しました。さらにMCPツール検索が自動モードでデフォルト有効化され、コンテキストウィンドウ使用量が削減され、OAuth URLが`console.anthropic.com`から`platform.claude.com`に変更されています。

  🔗 https://github.com/anthropics/claude-code/releases/tag/v2.1.7

- **v2.1.6**
  Claude Code v2.1.6では、複数の新機能と重要なバグ修正が実装されました。主な追加機能には、`/config`コマンドへの検索機能、`/doctor`コマンドへの更新情報セクション、`/stats`コマンドへの日付範囲フィルタリング、ネストされたスキルディレクトリの自動検出が含まれます。さらに、コンテキストウィンドウ使用率の表示フィールド追加、シェルコマンドの権限回避脆弱性の修正、ファイル修正判定の誤検知解決など、セキュリティとUXの両面で重要な改善が行われました。

  🔗 https://github.com/anthropics/claude-code/releases/tag/v2.1.6

- **v2.1.5**
  Claude-Code v2.1.5 は、2025年1月12日にリリースされました。このリリースの主な変更点は、内部一時ファイルに使用される一時ディレクトリをオーバーライドするための `CLAUDE_CODE_TMPDIR` 環境変数を追加したことです。この機能は、カスタム一時ディレクトリ要件を持つ環境において特に有用です。

  🔗 https://github.com/anthropics/claude-code/releases/tag/v2.1.5

- **v2.1.4**
  このリリースでは、2つの主な改善が加えられました。1つ目は、`CLAUDE_CODE_DISABLE_BACKGROUND_TASKS`環境変数の追加で、これにより自動バックグラウンド実行とCtrl+Bショートカットを含むすべてのバックグラウンドタスク機能を無効化できるようになりました。2つ目は、「ヘルプ改善」設定の取得機能の修正で、古いOAuthトークンによる失敗時にOAuthを更新して再試行するようになり、より堅牢になりました。

  🔗 https://github.com/anthropics/claude-code/releases/tag/v2.1.4

- **v2.1.3**
  このリリースでは、スラッシュコマンドとスキルの統合によって、メンタルモデルが簡潔になりました。主な改善点として、`/config`にリリースチャネル（`stable`または`latest`）の切り替え機能が追加され、到達不可能なパーミッションルールの検出と警告機能が実装されました。また、`/clear`コマンド後のプランファイル永続化の問題や、大きなiノード値を持つファイルシステムでのスキル重複検出の誤検出が修正されました。

  🔗 https://github.com/anthropics/claude-code/releases/tag/v2.1.3
