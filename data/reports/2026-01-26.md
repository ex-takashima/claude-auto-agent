### GitHub Blog

- **Build an agent into any app with the GitHub Copilot SDK**
  GitHubは、カスタムアプリケーションにエージェント型AI機能を組み込めるGitHub Copilot SDKのテクニカルプレビューを公開しました。このSDKは、GitHub Copilot CLIを動かす実証済みの実行ループへのプログラマティックアクセスを提供し、プランニング、ツール呼び出し、マルチターン実行などの機能を開発者がゼロから構築することなく利用できます。Node.js、Python、Go、.NETをサポートし、複数のAIモデル、カスタムツール定義、MCPサーバー、GitHub認証と統合可能で、YouTubeチャプター生成ツールやデスクトップ音声コマンドワークフローなど、多様なユースケースでの活用が始まっています。

  🔗 https://github.blog/news-insights/company-news/build-an-agent-into-any-app-with-the-github-copilot-sdk/

- **A cheat sheet to slash commands in GitHub Copilot CLI**
  GitHub Copilot CLIのスラッシュコマンドに関する包括的なガイドが公開されました。これらのコマンドは、自然言語プロンプトを使わずに特定のアクションをトリガーするプレフィックス付き命令で、「スピードと予測可能性」を提供します。セッション管理（`/clear`、`/session`）、ディレクトリアクセス（`/add-dir`、`/list-dirs`）、設定（`/model`、`/theme`）、外部サービス（`/delegate`、`/share`）の4つのカテゴリに整理され、ツール切り替えの削減、明示的なファイルアクセス境界によるセキュリティ向上、キーボード駆動ワークフローのアクセシビリティサポートなど、開発者の生産性向上に貢献します。

  🔗 https://github.blog/ai-and-ml/github-copilot/a-cheat-sheet-to-slash-commands-in-github-copilot-cli/

- **AI-supported vulnerability triage with the GitHub Security Lab Taskflow Agent**
  GitHub Security LabがLLMを活用した脆弱性トリアージ自動化フレームワークを開発し、8月以降約30件の実際の脆弱性を特定しました。このシステムは「タスクフロー」と呼ばれるYAML定義のタスクシーケンスを使用して、GitHub ActionsとJavaScriptプロジェクトのセキュリティ問題を体系的に分析します。従来のパターンマッチングツールでは対処が難しい誤検知の識別に特に効果的で、複雑な分析を小さな独立したタスクに分割することでハルシネーションを最小化しながら、実行時検証なしで精度を維持します。この手法は、従来のプログラミング手法では困難な意味的コード分析を必要とする反復的な多段階プロセスを伴う他の領域にも拡張可能です。

  🔗 https://github.blog/security/ai-supported-vulnerability-triage-with-the-github-security-lab-taskflow-agent/

- **Context windows, Plan agent, and TDD: What I learned building a countdown app with GitHub Copilot**
  Chris Reddingtonが GitHub Copilotを使って新年カウントダウンアプリを構築した実践例が紹介されています。彼は新しいチャットセッションを開始してコンテキストウィンドウを管理し、Planエージェントを使って曖昧な要件を質問を通じて明確化し、テスト駆動開発（TDD）を採用して年のロールオーバーなどのエッジケースのバグを発見しました。プロジェクトは基本的なカウントダウンタイマーから、花火やGitHub contribution graphビジュアライゼーションなど複数のテーマを持つ機能へと進化し、「コンテキストウィンドウ管理はスキルである」こと、カスタムエージェントが専門知識を提供すること、テスト実践を維持しながら不完全な初期実装を受け入れる反復的開発の重要性が強調されています。

  🔗 https://github.blog/developer-skills/application-development/context-windows-plan-agent-and-tdd-what-i-learned-building-a-countdown-app-with-github-copilot/

## Claude Code

### Claude Code GitHub Releases

- **v2.1.19**
  このリリースでは、古いタスクシステムに戻すための`CLAUDE_CODE_ENABLE_TASKS`環境変数の追加、カスタムコマンドでの引数アクセスのための短縮構文（`$0`、`$1`など）の実装、VSCodeユーザー向けのセッションフォーク機能の有効化など、いくつかの重要な機能が導入されました。バグ修正では、AVX命令サポートがないプロセッサでのクラッシュ、異なるディレクトリから再開した際の`/rename`と`/tag`コマンドの不具合、プロンプトスタッシュ使用時の貼り付けテキストの消失、ファイル書き込みプレビューでの空行省略など、複数の問題が解決されました。

  🔗 https://github.com/anthropics/claude-code/releases/tag/v2.1.19

- **v2.1.17**
  v2.1.17はメンテナンスリリースで、互換性と安定性の向上に焦点を当てています。主な変更点は、AVX命令サポートがないプロセッサでのクラッシュに対するバグ修正で、より幅広いハードウェアプラットフォームでツールを実行できるようにしました。これは、Advanced Vector Extensions（AVX）サポートを欠く古いまたは一般的でないCPUアーキテクチャを持つユーザーにとって重要な修正でした。新機能は導入されておらず、純粋に安定性に焦点を当てたアップデートです。

  🔗 https://github.com/anthropics/claude-code/releases/tag/v2.1.17

- **v2.1.16**
  このリリースでは、依存関係追跡機能を備えた新しいタスク管理システムが導入され、プロジェクトの整理とワークフローの調整が強化されました。注目すべき改善点として、VSCode向けのネイティブプラグイン管理サポート、OAuthユーザーがSessionsダイアログから直接リモートClaudeセッションをブラウズして再開できる機能などがあります。サブエージェントの使用量が多いセッション再開時のメモリ不足クラッシュの修正や、Windows上でClaude Codeサイドバーが起動時に表示されない問題を引き起こした競合状態の修正など、重要な安定性の問題が解決されました。

  🔗 https://github.com/anthropics/claude-code/releases/tag/v2.1.16
