## 技術ブログ

### GitHub Blog

- **Building an agentic memory system for GitHub Copilot**

  GitHub は Copilot に「クロスエージェントメモリシステム」を導入しました。このシステムにより、複数のエージェント（コーディング、CLI、コードレビュー）が開発ワークフロー全体で学習と改善を重ねることができます。メモリシステムの核となるのは「just-in-time verification」という仕組みです。「保存されたメモリは引用元コード箇所への参照を含み、使用時にリアルタイムで検証される」ことで、古い情報や誤った情報を避けています。評価結果では、Copilot コーディングエージェントでプルリクエストのマージ率が 7% 向上し、コードレビューの正確性が 3%、再現率が 4% それぞれ改善されました。

  🔗 https://github.blog/ai-and-ml/github-copilot/building-an-agentic-memory-system-for-github-copilot/

- **When protections outlive their purpose: A lesson on managing defense systems at scale**

  GitHubは、過去のインシデント対応時に実装された古い保護メカニズムが、正当なユーザーをブロックしていた問題を報告しました。「ユーザーが通常のブラウジング中に『リクエストが多すぎます』エラーを受け取っていた」ことが判明し、調査の結果、複合的なフィンガープリンティング手法を使用したルールが誤検知を引き起こしていたことが明らかになりました。全トラフィックの約0.003-0.004%に影響していましたが、GitHubはこれらの古い保護ルールを削除し、将来的には保護メカニズムのライフサイクル管理を改善する方針を示しています。

  🔗 https://github.blog/engineering/infrastructure/when-protections-outlive-their-purpose-a-lesson-on-managing-defense-systems-at-scale/

- **Community-powered security with AI: an open source framework for security research**

  GitHubセキュリティラボは、AIを活用した「GitHub Security Lab Taskflow Agent」という新しいオープンソースフレームワークを発表しました。このフレームワークは、「コミュニティが力を合わせてセキュリティを向上させる」という同チームの基本理念に基づいています。ユーザーは「タスクフロー」と呼ばれるYAML形式の設定ファイルを使用して、セキュリティ研究の自動化ワークフローを定義できます。同フレームワークはModel Context Protocol（MCP）インターフェースを活用し、CodeQLなど既存のセキュリティツールとの統合を可能にしています。

  🔗 https://github.blog/security/community-powered-security-with-ai-an-open-source-framework-for-security-research/

- **What AI is actually good for, according to developers**

  本記事は、AIコーディングツールが開発者にとって本当に有用であるべきことについて論じています。GitHub開発者アドボケートのCassidy Williamsと、Visual StudioシニアプロダクトマネージャーのDalia Abo Sheashaが、開発者の「フロー状態」を保つことの重要性を強調しています。要点として、「AI is designed to help you do what you love most, not replace your expertise（AIは皆さんが最も愛することをサポートするためのもので、専門知識の代替ではない）」と述べられています。効果的なAIツールは、反復的なタスクを削減しながらも開発者の判断を尊重し、チャット機能よりもコンテキストに応じた提案が有効であることが強調されています。

  🔗 https://github.blog/ai-and-ml/generative-ai/what-ai-is-actually-good-for-according-to-developers/
